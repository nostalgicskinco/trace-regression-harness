# Contributing

We welcome contributions! By submitting a pull request, you agree to the following CLA:

## Contributor License Agreement

By contributing to this project, you grant the project maintainers a
**perpetual, worldwide, non-exclusive, royalty-free, irrevocable license** to:

1. Use, reproduce, modify, and distribute your contributions under the AGPL-3.0-or-later license
2. Relicense your contributions under alternative commercial licenses

This ensures the project can offer dual licensing (AGPL + Commercial) while
incorporating community contributions.

## How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/my-feature`
3. Write tests for new functionality
4. Ensure `go test ./...` passes
5. Submit a pull request with a clear description

## Code Standards

- Follow Go conventions (`gofmt`, `go vet`)
- Add tests for all new rules and features
- Keep policy rules composable and well-documented
- Update example policies when adding new rule types
